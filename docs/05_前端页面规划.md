## 6. 关键页面字段速查

### 6.1 学生管理
**查询字段**：关键词（学号/姓名）、年级、专业、班级、状态
**表格字段**：学号、姓名、性别、年级、专业、班级、电话、状态、入学日期、操作
**表单字段**：学号、姓名、性别、出生日期、年级、专业、班级、电话、地址、入学日期、状态、备注

### 6.2 成绩管理
**查询字段**：学期、课程、班级、学号/姓名、等级
**表格字段**：学号、姓名、班级、课程、学期、平时分、期末分、总评、等级、录入教师、操作
**表单字段**：学生、课程、学期、平时成绩、期末成绩（总评自动计算）

### 6.3 考勤管理
**查询字段**：日期范围、课程、班级、学号/姓名、状态
**表格字段**：日期、学号、姓名、班级、课程、状态、记录人、备注、操作
**表单字段**：学生、课程、日期、状态、备注

### 6.4 通知公告
**查询字段**：关键词、分类、状态（admin）、是否置顶
**表格字段**：标题、分类、状态、置顶、发布人、时间、浏览次数、操作
**表单字段**：标题、分类、内容、状态、置顶、发布时间、过期时间（可选）

## 7. 权限控制规则

### 7.1 页面权限
- 通过路由 `meta.roles` 控制访问
- 路由守卫检查 token 和角色
- 无权限跳转 `/403`

### 7.2 按钮权限
- 根据角色显示/隐藏操作按钮
- 学生：只读（个人相关）
- 教师：可管理（学生、班级、课程、成绩、考勤、奖惩），不可访问系统管理
- 管理员：全部权限

### 7.3 数据权限
- 教师只能看到自己教授的课程、班级、学生数据
- 学生只能看到个人数据
- 管理员可查看全校数据

## 8. 全局交互规范

### 8.1 表格页统一结构
```
页面标题 → 查询表单 → 操作按钮 → 数据表格 → 分页 → 新增/编辑弹窗 → 详情抽屉
```

### 8.2 操作流程
- **查询**：输入条件 → 点击查询 → 刷新列表；点击重置 → 清空条件 → 刷新
- **新增**：点击新增 → 打开弹窗 → 填写表单 → 提交 → 成功则关闭并刷新
- **编辑**：点击编辑 → 填充表单 → 修改 → 提交 → 成功则关闭并刷新
- **删除**：点击删除 → 确认弹窗 → 调用删除接口 → 成功则刷新列表

### 8.3 状态标签颜色
| 状态 | 颜色 |
|---|---|
| 正常/在读/启用/已发布 | success（绿） |
| 迟到/早退/中等/良好 | warning（橙） |
| 请假/休学/中等 | info（蓝） |
| 缺勤/退学/不及格/禁用 | danger（红） |

## 10. 工具与配置

### 10.1 枚举映射（`src/utils/enums.js`）
统一维护所有枚举的中文映射，如：
- 角色：admin/teacher/student
- 学籍状态：studying/suspended/withdrawn/graduated
- 成绩等级：excellent/good/medium/pass/fail
- 考勤状态：normal/late/leave_early/leave/absent
- 奖惩类型：reward/punishment
- 公告状态：draft/published
- 公告分类：通知/新闻/活动

### 10.2 Axios 封装（`src/api/request.js`）
- 设置 `baseURL: '/api'`
- 请求拦截器自动添加 `Authorization: Bearer ${token}`
- 响应拦截器统一处理 `code !== 200` 错误
- 统一处理 401/403/500 错误提示
- 返回 `res.data` 减少页面重复代码

### 10.3 API 模块拆分
每个业务模块一个文件：
```text
src/api/
  auth.js       # 登录/登出
  students.js   # 学生
  teachers.js   # 教师
  majors.js     # 专业
  classes.js    # 班级
  courses.js    # 课程
  scores.js     # 成绩
  attendance.js # 考勤
  rewardsPunishments.js  # 奖惩
  notices.js    # 公告
  users.js      # 用户
  profile.js    # 个人中心
  logs.js       # 日志
  options.js    # 下拉选项
```

### 10.4 Pinia Store
- `stores/user.js`：管理 `token`、`userInfo`、`role`、`isLoggedIn`、`login()`、`logout()`
- 使用 `localStorage` 持久化 token 和用户信息

## 13. 前端验收标准

### 13.1 基础框架
- 项目正常启动
- 登录页可访问
- 后台布局正常
- 菜单和路由切换正常
- 刷新页面无报错

### 13.2 登录权限
- 正确账号可登录
- 错误账号提示错误
- 未登录跳转登录页
- 不同角色显示不同菜单
- 无权限跳转 403
- token 失效跳转登录页

### 13.3 页面功能
- 所有管理页面可查询、新增、编辑、删除
- 成绩可录入和统计
- 考勤可登记和统计
- 公告可发布
- 个人中心可查看信息、修改密码
- 首页看板显示统计和图表

### 13.4 交互体验
- 表单必填校验有效
- 删除有确认框
- 保存成功/失败有提示
- 列表有 loading
- 空数据有提示
- 分页、搜索、弹窗正常
- 状态标签显示正确
