# 03_数据库设计.md

# 本地学校学生管理网页仿真系统：数据库设计（精简版）

## 1. 基本信息

| 项目 | 内容 |
|---|---|
| 数据库 | SQLite |
| ORM | Prisma |
| 数据库文件 | `backend/prisma/dev.db` |

---

## 2. 命名规范

- **表名**：英文小写复数，下划线连接（如 `attendance_records`）
- **字段名**：驼峰命名（Prisma 风格，如 `studentNo`、`createdAt`）
- **主键**：统一使用 `id`（自增整数）
- **时间字段**：`createdAt`、`updatedAt`（必含），`deletedAt`（第一版暂不使用）

---

|---|---|---|

| students | 学生表 | ✓ |

## 4. 实体关系

```text
users
  ├── teacherId -> teachers.id
  └── studentId -> students.id

majors
  └── classes
        └── students

teachers
  ├── classes.headTeacherId
  └── courses.teacherId

courses
  ├── scores
  ├── attendance_records
  └── course_classes

students
  ├── scores
  ├── attendance_records
  └── rewards_punishments
```

## 7. Prisma Schema

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = "file:./dev.db"
}

model User {
  id          Int       @id @default(autoincrement())
  username    String    @unique
  password    String
  nickname    String?
  role        String    @default("student")
  status      String    @default("enabled")
  teacherId   Int?
  studentId   Int?
  lastLoginAt DateTime?

  teacher     Teacher?  @relation(fields: [teacherId], references: [id])
  student     Student?  @relation(fields: [studentId], references: [id])
  notices     Notice[]  @relation("NoticePublisher")
  logs        OperationLog[]

  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}

model Student {
  id              Int                  @id @default(autoincrement())
  studentNo       String               @unique
  name            String
  gender          String               @default("unknown")
  birthDate       DateTime?
  phone           String?
  address         String?
  grade           String
  enrollmentDate  DateTime             @default(now())
  status          String               @default("studying")
  avatar          String?
  remark          String?

  majorId         Int
  classId         Int

  major           Major                @relation(fields: [majorId], references: [id])
  class           Class                @relation(fields: [classId], references: [id])
  user            User[]
  scores          Score[]
  attendance      AttendanceRecord[]
  rewardPunishes  RewardPunishment[]

  createdAt       DateTime             @default(now())
  updatedAt       DateTime             @updatedAt
}

model Teacher {
  id              Int                  @id @default(autoincrement())
  teacherNo       String               @unique
  name            String
  gender          String               @default("unknown")
  department      String
  title           String?
  phone           String?
  email           String?
  isHeadTeacher   Boolean              @default(false)
  remark          String?

  user            User[]
  headClasses     Class[]              @relation("ClassHeadTeacher")
  courses         Course[]
  recordedScores  Score[]              @relation("ScoreRecorder")
  attendance      AttendanceRecord[]   @relation("AttendanceRecorder")

  createdAt       DateTime             @default(now())
  updatedAt       DateTime             @updatedAt
}

model Major {
  id          Int       @id @default(autoincrement())
  majorNo     String    @unique
  name        String
  department  String
  duration    Int       @default(4)
  description String?
  status      String    @default("enabled")

  classes     Class[]
  students    Student[]
  courses     Course[]

  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}

model Class {
  id             Int           @id @default(autoincrement())
  classNo        String        @unique
  name           String
  grade          String
  majorId        Int
  headTeacherId  Int?
  status         String        @default("enabled")
  remark         String?

  major          Major         @relation(fields: [majorId], references: [id])
  headTeacher    Teacher?      @relation("ClassHeadTeacher", fields: [headTeacherId], references: [id])
  students       Student[]
  courseClasses  CourseClass[]

  createdAt      DateTime      @default(now())
  updatedAt      DateTime      @updatedAt
}

model Course {
  id              Int                  @id @default(autoincrement())
  courseNo        String               @unique
  name            String
  type            String               @default("required")
  credit          Float                @default(0)
  teacherId       Int
  majorId         Int?
  grade           String
  semester        String
  status          String               @default("enabled")
  remark          String?

  teacher         Teacher              @relation(fields: [teacherId], references: [id])
  major           Major?               @relation(fields: [majorId], references: [id])
  scores          Score[]
  attendance      AttendanceRecord[]
  courseClasses   CourseClass[]

  createdAt       DateTime             @default(now())
  updatedAt       DateTime             @updatedAt
}

model CourseClass {
  id        Int      @id @default(autoincrement())
  courseId  Int
  classId   Int

  course    Course   @relation(fields: [courseId], references: [id])
  class     Class    @relation(fields: [classId], references: [id])

  createdAt DateTime @default(now())

  @@unique([courseId, classId])
}

model Score {
  id          Int       @id @default(autoincrement())
  studentId   Int
  courseId    Int
  semester    String
  usualScore  Float     @default(0)
  finalScore  Float     @default(0)
  totalScore  Float     @default(0)
  level       String    @default("fail")
  recorderId  Int?
  remark      String?

  student     Student   @relation(fields: [studentId], references: [id])
  course      Course    @relation(fields: [courseId], references: [id])
  recorder    Teacher?  @relation("ScoreRecorder", fields: [recorderId], references: [id])

  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  @@unique([studentId, courseId, semester])
}

model AttendanceRecord {
  id              Int       @id @default(autoincrement())
  studentId       Int
  courseId        Int
  attendanceDate  DateTime
  status          String    @default("normal")
  recorderId      Int?
  remark          String?

  student         Student   @relation(fields: [studentId], references: [id])
  course          Course    @relation(fields: [courseId], references: [id])
  recorder        Teacher?  @relation("AttendanceRecorder", fields: [recorderId], references: [id])

  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt

  @@unique([studentId, courseId, attendanceDate])
}

model RewardPunishment {
  id              Int       @id @default(autoincrement())
  studentId        Int
  type             String    @default("reward")
  title            String
  level            String?
  recordDate       DateTime  @default(now())
  description      String?
  recorderUserId   Int?

  student          Student   @relation(fields: [studentId], references: [id])
  recorderUser     User?     @relation(fields: [recorderUserId], references: [id])

  createdAt        DateTime  @default(now())
  updatedAt        DateTime  @updatedAt
}

model Notice {
  id          Int       @id @default(autoincrement())
  title       String
  content     String
  category    String    @default("system")
  status      String    @default("draft")
  isTop       Boolean   @default(false)
  publisherId Int?
  publishedAt DateTime?
  expiredAt   DateTime?
  viewCount   Int       @default(0)

  publisher   User?     @relation("NoticePublisher", fields: [publisherId], references: [id])

  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
}

model OperationLog {
  id          Int       @id @default(autoincrement())
  userId      Int?
  username    String?
  role        String?
  module      String
  action      String
  description String?
  result      String    @default("success")
  ip          String?

  user        User?     @relation(fields: [userId], references: [id])

  createdAt   DateTime  @default(now())
}
```

| teachers | teacherNo |

| courses | courseNo |

| 表 | 字段 | 用途 |

## 9. 数据完整性核心规则

| 表 | 关键规则 |
|---|---|
| users | username唯一、密码bcrypt加密、角色固定值、禁用不可登录 |
| students | studentNo唯一、必须关联专业和班级、学籍状态固定值 |
| teachers | teacherNo唯一、已关联课程/班级不建议直接删除 |
| classes | classNo唯一、必须关联专业、人数从学生表统计 |
| courses | courseNo唯一、必须关联教师、学分≥0 |
| scores | 分数0-100、同一学生+课程+学期唯一、总评和等级自动计算 |
| attendance | 同一学生+课程+日期唯一、状态固定值 |
| rewards_punishments | 必须关联学生、type固定值 |

---

## 10. 删除策略

- 第一版采用**物理删除**，删除前校验关联数据
- 有强关联数据时禁止删除（如专业下有班级、班级下有学生、课程下有成绩）
- 后续可改为逻辑删除（通过 `status` 字段停用）

## 12. 数据库迁移与种子脚本

```bash
# 初始化
cd backend
npx prisma init
npx prisma migrate dev --name init
npx prisma generate

# 修改schema后
npx prisma migrate dev --name update_schema
npx prisma generate

# 重置数据库
npx prisma migrate reset
```

种子脚本位置：`backend/prisma/seed.js`

`package.json` 配置：
```json
{
  "prisma": {
    "seed": "node prisma/seed.js"
  }
}
```

运行种子：`npx prisma db seed`

## 13. 开发注意事项

1. **不使用真实隐私数据**——只用模拟姓名和数据
2. **字段不过度复杂**——不加身份证、银行卡等敏感字段
3. **枚举值前后端统一**——建议后端统一返回或前端维护映射
4. **先保证核心功能**——优先完成8张必做表
