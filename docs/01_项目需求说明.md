# 01_项目需求说明

## 项目概述

| 项目 | 内容 |
|---|---|
| 项目名称 | 本地学校学生管理网页仿真系统 |
| 项目类型 | 本科毕业设计 / 本地 Web 仿真系统 |
| 技术栈 | Vue 3 + Vite + Element Plus + Node.js + Express + Prisma + SQLite |
| 运行方式 | 本地运行，不部署公网 |
| 用户角色 | 管理员(admin)、教师(teacher)、学生(student) |

## 用户角色与权限

**管理员**：全部管理权限 — 学生/教师/专业/班级/课程/成绩/考勤/奖惩/公告/用户管理、操作日志、首页看板

**教师**：查看个人信息、所授课程、相关学生；录入所授课程成绩和考勤；查看公告

**学生**：查看个人信息、个人成绩/考勤/奖惩/课程；查看公告；修改密码

## 核心模块

| 模块 | 优先级 | 说明 |
|---|---|---|
| 登录与权限 | 必做 | JWT认证，角色菜单控制 |
| 首页看板 | 必做 | 统计卡片 + ECharts图表 |
| 学生管理 | 必做 | CRUD + 搜索筛选分页 |
| 教师管理 | 必做 | CRUD + 搜索筛选分页 |
| 专业管理 | 必做 | CRUD |
| 班级管理 | 必做 | CRUD + 分配班主任 |
| 课程管理 | 必做 | CRUD + 分配教师 |
| 成绩管理 | 必做 | 录入成绩，自动计算总评和等级 |
| 考勤管理 | 必做 | 登记考勤，状态管理 |
| 奖惩管理 | 建议做 | 奖励/处分记录 |
| 通知公告 | 建议做 | 发布/查看公告 |
| 用户管理 | 建议做 | 账号管理、重置密码 |
| 操作日志 | 可选 | 关键操作记录 |
| 导入导出 | 可选 | Excel导入导出 |

## 核心业务规则

**成绩计算**：总评 = 平时成绩×40% + 期末成绩×60%
等级：90-100优秀、80-89良好、70-79中等、60-69及格、0-59不及格

**考勤状态**：正常、迟到、早退、请假、缺勤

**课程类型**：必修、选修、公共课、实践课

**学籍状态**：在读、休学、退学、毕业

## 系统边界

**实现**：本地Web后台、三类角色登录、基础数据CRUD、成绩/考勤业务、首页看板、本地SQLite存储、模拟数据初始化

**不实现**：公网部署、微信小程序、短信/邮件、人脸识别、复杂排课、微服务、真实数据对接

## 数据要求

全模拟数据，建议规模：管理员1个、教师5-10个、学生30-100个、专业3-5个、班级6-10个、课程10-20门、成绩100-300条、考勤100-300条、公告5-10条

## 项目目录

```
student-management-system/
  frontend/          # Vue 3 + Vite 前端
  backend/           # Node.js + Express 后端
    prisma/          # Prisma schema + seed
    src/             # routes/controllers/middlewares
  docs/              # 设计文档
  README.md
```

## 开发原则

1. 先可运行，再完善功能，最后优化展示
2. 按模块逐步开发，不一次性生成全部代码
3. 前后端分离，后端接口优先
4. 前端权限仅用于交互优化，最终权限由后端控制
5. 不使用真实学生隐私数据
