# 本地学校学生管理网页仿真系统：开发阶段任务（精简版）

---

## 1. 总体开发原则

1. **先跑通，再完善**：优先实现完整运行流程，而不是追求所有功能
2. **按模块开发**：每次只开发一个明确模块，避免 AI 一次性生成过多代码
3. **后端优先**：数据库 → 后端接口 → 前端页面 → 联调测试
4. **控制范围**：不做公网部署、人脸识别、短信、支付、复杂排课、移动 App

## 3. 各阶段核心任务速查

### 3.1 第 1 阶段：项目骨架
**后端**：Express 项目、cors/dotenv 配置、`/api/health` 接口、目录结构（routes/controllers/middlewares/utils）
**前端**：Vue 3 + Vite、Element Plus、Vue Router、Pinia、Axios、ECharts、Login.vue、Dashboard.vue
**验收**：前后端都能启动，端口 3000/5173 不冲突

### 3.2 第 2 阶段：数据库
**任务**：Prisma + SQLite、编写 `schema.prisma`（11 个核心模型）、编写 `seed.js` 模拟数据
**数据量**：管理员 1、教师 2-5、学生 30-100、专业 3-5、班级 4-8、课程 10-20、成绩/考勤 50-200、公告 5-10
**验收**：数据库生成成功、种子数据插入成功、Prisma Studio 可查看数据、密码 bcrypt 加密

### 3.3 第 3 阶段：登录权限
**后端**：JWT、bcrypt、登录/当前用户/修改密码接口、auth/role 中间件、统一响应格式、401/403 处理
**前端**：Axios 封装、Pinia user store、登录页、后台布局、路由守卫、动态菜单、退出登录
**验收**：三类角色可登录、错误密码不可登录、未登录跳转登录页、不同角色显示不同菜单、无权限跳转 403

### 3.4 第 4 阶段：基础数据管理（5 个模块）
**开发顺序**：专业 → 教师 → 班级 → 学生 → 课程（考虑依赖关系）
**每个模块包含**：
- 后端：CRUD 接口 + 唯一校验 + 关联校验
- 前端：列表页（查询/表格/分页/弹窗）+ API 封装
- 权限：管理员全权限，教师/学生只读或不可访问
**验收**：所有 CRUD 正常、分页搜索正常、删除有确认、编号唯一校验生效、角色权限正确

### 3.5 第 5 阶段：核心业务（4 个模块）
**开发顺序**：成绩 → 考勤 → 公告 → 奖惩
**关键业务规则**：
- 成绩：总评 = 平时×40% + 期末×60%（后端自动计算）、等级自动生成、同一学生同一课程同一学期不能重复录入
- 考勤：同一学生同一课程同一天不能重复登记、状态包含正常/迟到/早退/请假/缺勤
- 公告：教师和学生只能查看已发布公告、置顶优先
- 奖惩：学生只能查看个人、教师只能查看相关学生
**验收**：业务规则正确、权限控制正确、统计功能正常

### 3.6 第 6 阶段：看板与个人中心
**首页看板**：统计卡片（学生/教师/班级/课程总数、今日考勤异常）、图表（各年级人数柱状图、男女比例饼图、成绩分布）、最近公告、快捷入口（按角色）
**个人中心**：根据角色显示不同信息（管理员：账号信息；教师：教师信息+授课课程；学生：学生信息+成绩/考勤/奖惩摘要）+ 修改密码
**用户管理**（admin）：账号 CRUD + 禁用 + 重置密码
**操作日志**（admin，可选）：查看关键操作日志
**验收**：图表正常显示、个人中心信息完整、用户管理可用、日志记录完整

### 3.7 第 7 阶段：测试优化
**功能测试**：所有 CRUD 操作、登录退出、首页看板、个人中心
**权限测试**：三类角色权限是否正确、无权限是否跳转 403
**接口测试**：401/403 返回、唯一约束、数据范围、分页参数
**体验优化**：loading、空状态、成功/失败提示、删除确认框、表单校验、状态标签、菜单图标
**验收**：系统稳定、核心功能无报错、权限正确、演示流程顺畅

### 3.8 第 8 阶段：毕设包装
**整理内容**：
1. README：项目简介、技术栈、功能、目录结构、运行步骤、演示账号、系统截图、常见问题、后续优化
2. 运行部署说明：环境要求、依赖安装、数据库迁移、启动步骤、报错处理
3. 测试用例文档
4. 项目截图（至少 14 张：登录页、各管理页面、学生端页面、个人中心、数据库等）
5. 演示账号：admin/123456、teacher001/123456、student001/123456
6. 答辩演示流程（推荐 13 步流程）
7. 论文材料整理
**验收**：按 README 可启动项目、演示流程顺畅、文档齐全、适合毕业设计答辩

## 4. 任务优先级划分

### 4.1 必须完成（系统完整）
- 项目启动、数据库、登录权限
- 学生/教师/专业/班级/课程管理
- 成绩/考勤管理
- 首页看板、个人中心
- README、运行说明

### 4.2 建议完成（增强展示效果）
- 通知公告、奖惩管理
- 用户管理、操作日志
- 成绩/考勤统计
- 详情页（学生/班级/课程）

### 4.3 可选完成（时间充足再做）
- Excel 导入导出
- 富文本编辑器
- 图表美化
- 头像上传
- 数据备份恢复
- 主题切换、多语言
- 移动端适配

### 4.4 不建议完成（写入未来展望）
- 公网部署、人脸识别、短信验证码
- 真实邮件通知、在线支付
- 复杂排课算法、微服务、Docker
- 真实教务系统对接、真实学生数据

## 9. 常见风险与应对

| 风险 | 表现 | 解决 |
|---|---|---|
| AI 一次性改太多文件 | 已有代码被破坏 | 每次只开发一个模块，明确"不修改其他模块" |
| 数据库和接口字段不一致 | 前端拿不到数据 | 严格参考 03/04/05 三份文档，不随意改字段 |
| 权限只做前端隐藏 | 用户手动访问接口仍能拿到数据 | 前端隐藏 + 后端接口必须校验角色 + 数据范围过滤 |
| 功能越做越多 | 项目延期、代码复杂 | 优先必做功能，可选功能写入未来展望 |
| 缺少演示数据 | 首页图表为空、展示效果差 | 编写 seed.js，准备足量模拟数据 |

## 10. 答辩演示流程（推荐）

1. 打开登录页
2. 使用管理员账号登录
3. 展示首页数据看板
4. 展示学生管理：查询、新增、编辑
5. 展示课程管理：查看课程和任课教师
6. 展示成绩管理：录入成绩并展示自动计算
7. 展示考勤管理：登记考勤
8. 展示通知公告：发布公告
9. 退出管理员账号
10. 使用学生账号登录
11. 展示学生查看个人成绩、考勤和公告
12. 简要展示数据库保存的数据
13. 说明系统只在本地运行，使用模拟数据
