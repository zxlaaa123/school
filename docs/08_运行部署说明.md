## 2. 文档目的

本文档用于说明本地学校学生管理网页仿真系统的安装、配置、数据库初始化、前后端启动、演示账号、常见问题和交付检查流程。

本文档适合以下人员使用：

1. 项目开发者。
2. 使用 Trae、CodeBuddy 等 AI 编程工具的开发者。
3. 毕业设计答辩学生。
4. 项目测试人员。
5. 需要在本地电脑运行该系统的人员。

本系统仅用于本地毕业设计演示，不需要部署到公网服务器。

---

## 3. 项目说明

本项目是一个本地运行的学校学生管理网页仿真系统，用于模拟学校在日常教学管理中对学生、教师、专业、班级、课程、成绩、考勤、奖惩和公告等信息进行管理。

系统特点：

1. 前后端分离。
2. 本地 Web 页面运行。
3. 使用 SQLite 本地数据库。
4. 使用模拟数据。
5. 支持管理员、教师、学生三类角色。
6. 支持成绩、考勤、公告、统计看板等功能。
7. 适合本科毕业设计展示和论文编写。

# 9. 后端运行说明

---

## 9.1 进入后端目录

在项目根目录执行：

```bash
cd backend
```

---

## 9.2 安装后端依赖

执行：

```bash
npm install
```

如果项目尚未安装依赖，建议安装以下包：

```bash
npm install express cors dotenv jsonwebtoken bcrypt @prisma/client
npm install prisma nodemon --save-dev
```

---

## 9.3 后端 package.json 脚本建议

建议 `backend/package.json` 中包含：

```json
{
  "scripts": {
    "dev": "nodemon src/app.js",
    "start": "node src/app.js",
    "prisma:studio": "npx prisma studio",
    "prisma:migrate": "npx prisma migrate dev",
    "prisma:seed": "npx prisma db seed",
    "prisma:reset": "npx prisma migrate reset"
  }
}
```

---

## 9.4 配置环境变量

在 `backend` 目录下创建 `.env` 文件：

```text
DATABASE_URL="file:./dev.db"
JWT_SECRET="student_management_secret_key"
JWT_EXPIRES_IN="7d"
PORT=3000
```

说明：

| 配置项 | 说明 |
|---|---|
| DATABASE_URL | SQLite 数据库连接地址 |
| JWT_SECRET | JWT 签名密钥 |
| JWT_EXPIRES_IN | Token 有效期 |
| PORT | 后端服务端口 |

如果使用 Prisma 默认 SQLite 路径，`DATABASE_URL` 通常写在 `backend/prisma` 相对路径中：

```text
DATABASE_URL="file:./dev.db"
```

---

## 9.5 初始化 Prisma

如果项目还没有初始化 Prisma，执行：

```bash
npx prisma init
```

该命令会生成：

```text
backend/prisma/schema.prisma
backend/.env
```

如果项目已经有 `schema.prisma`，则不需要重复执行。

---

## 9.6 执行数据库迁移

确保 `backend/prisma/schema.prisma` 已经按照数据库设计完成后，执行：

```bash
npx prisma migrate dev --name init
```

执行成功后，会生成 SQLite 数据库文件：

```text
backend/prisma/dev.db
```

同时会生成 Prisma Client。

---

## 9.7 生成 Prisma Client

如果修改了数据库模型，建议执行：

```bash
npx prisma generate
```

通常执行迁移时会自动生成，但手动执行也可以。

---

## 9.8 初始化种子数据

如果项目中已经编写 `backend/prisma/seed.js`，执行：

```bash
npx prisma db seed
```

种子数据建议包含：

1. 管理员账号。
2. 教师账号。
3. 学生账号。
4. 专业数据。
5. 班级数据。
6. 教师数据。
7. 学生数据。
8. 课程数据。
9. 成绩数据。
10. 考勤数据。
11. 公告数据。
12. 奖惩数据。

---

## 9.9 启动后端服务

执行：

```bash
npm run dev
```

启动成功后，终端应显示类似信息：

```text
Server is running at http://localhost:3000
```

---

## 9.10 测试后端健康接口

浏览器访问：

```text
http://localhost:3000/api/health
```

或使用接口工具请求：

```text

```

  "message": "服务正常",

## 10.3 前端 package.json 脚本建议

建议 `frontend/package.json` 中包含：

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  }
}
```

---

## 10.4 配置前端代理

为了避免跨域问题，建议在 `frontend/vite.config.js` 中配置代理。

示例：

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true
      }
    }
  }
})
```

这样前端请求 `/api/students` 会自动代理到：

```text
http://localhost:3000/api/students
```

# 14. 本地开发推荐启动顺序

推荐每次开发时按照以下顺序启动：

```text
1. 启动后端
2. 确认 /api/health 正常
3. 启动前端
4. 打开浏览器访问前端
5. 登录系统
```

原因：

1. 前端页面需要请求后端接口。
2. 后端未启动时，前端会出现接口请求失败。
3. 先确认后端正常，可以减少排错成本。

# 15. 数据库重置说明

如果开发过程中数据库数据混乱，可以重置数据库。

在 `backend` 目录执行：

```bash
npx prisma migrate reset
```

执行后会：

1. 删除当前数据库。
2. 重新执行迁移。
3. 重新执行种子数据脚本。
4. 清空之前手动添加的数据。

注意：

```text
该操作会清空当前数据库数据，请谨慎使用。
```

## 17.1 前端请求路径

```text

```

```js

```

```text

```

---

## 17.2 后端接口基础路径

后端所有接口建议以 `/api` 开头。

示例：

```text
POST /api/auth/login
GET /api/students
GET /api/courses
GET /api/scores
```

---

## 17.3 Token 携带方式

登录成功后，前端保存 token，并在请求头中携带：

```text
Authorization: Bearer token
```

如果未携带 token，后端应返回：

```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null
}
```

---

## 17.4 统一响应格式

  "code": 200,

  "data": {}

  "data": {

# 20. 常见问题与解决方案

---

## 20.1 npm install 很慢

### 问题表现

执行 `npm install` 很久没有完成。

### 可能原因

1. 网络较慢。
2. npm 源访问不稳定。

### 解决方案

可以尝试使用国内镜像源：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

恢复默认源：

```bash
npm config set registry https://registry.npmjs.org
```

---

## 20.2 端口被占用

### 问题表现

启动后端或前端时提示端口已被占用。

常见提示：

```text
EADDRINUSE: address already in use
```

### 解决方案

方案一：关闭占用端口的程序。

方案二：修改端口。

后端 `.env`：

```text
PORT=3001
```

前端 `vite.config.js`：

```js
server: {
  port: 5174
}
```

如果修改了后端端口，前端代理地址也要同步修改。

---

## 20.3 前端请求接口失败

### 问题表现

页面提示网络异常或接口请求失败。

### 可能原因

1. 后端未启动。
2. 前端代理配置错误。
3. 后端端口不是 3000。
4. 接口路径不一致。
5. Token 失效。

### 解决方案

1. 访问 `http://localhost:3000/api/health` 检查后端。
2. 检查 `vite.config.js` 代理配置。
3. 检查前端 API 文件请求路径。
4. 重新登录获取新 token。
5. 查看浏览器控制台 Network 请求信息。

---

## 20.4 Prisma migrate 报错

### 问题表现

执行迁移时报错。

### 可能原因

1. `schema.prisma` 语法错误。
2. `.env` 中 `DATABASE_URL` 配置错误。
3. 数据库文件被占用。
4. 之前迁移历史冲突。

### 解决方案

1. 检查 `schema.prisma`。
2. 检查 `.env`。
3. 关闭 Prisma Studio 或 SQLite 工具。
4. 开发环境可执行重置：

```bash
npx prisma migrate reset
```

注意：该命令会清空数据库。

---

## 20.5 npx prisma db seed 不执行

### 问题表现

执行种子命令时提示没有配置 seed。

### 解决方案

在 `backend/package.json` 中添加：

```json
{
  "prisma": {
    "seed": "node prisma/seed.js"
  }
}
```

然后执行：

```bash
npx prisma db seed
```

---

## 20.6 登录失败

### 问题表现

使用演示账号登录失败。

### 可能原因

1. 种子数据未初始化。
2. 密码被修改。
3. bcrypt 加密和校验逻辑不一致。
4. 用户状态为 disabled。

### 解决方案

1. 执行：

```bash
npx prisma db seed
```

2. 使用 Prisma Studio 检查 users 表。
3. 确认用户状态为 enabled。
4. 确认后端登录接口使用 bcrypt.compare 校验密码。

---

## 20.7 页面刷新后登录状态丢失

### 问题表现

登录后刷新页面，跳回登录页。

### 可能原因

1. token 没有保存到 localStorage。
2. Pinia 状态刷新后丢失。
3. 路由守卫未从 localStorage 恢复用户信息。

### 解决方案

1. 登录成功后保存 token：

```js
localStorage.setItem('token', token)
```

2. 页面初始化时从 localStorage 读取 token。
3. 路由守卫中检查 token。

---

## 20.8 学生能访问管理员页面

### 问题表现

学生手动输入 `/users` 可以进入用户管理页面。

### 可能原因

1. 前端路由守卫未检查角色。
2. 后端接口未检查角色。

### 解决方案

1. 路由 meta 中添加 roles。
2. 路由守卫判断当前角色。
3. 后端接口使用 role middleware。
4. 学生访问管理员接口返回 403。

---

## 20.9 图表不显示

### 问题表现

首页 ECharts 图表区域空白。

### 可能原因

1. ECharts 未安装。
2. DOM 容器没有高度。
3. 数据为空。
4. 图表初始化时机不对。

### 解决方案

1. 确认安装：

```bash
npm install echarts
```

2. 给图表容器设置高度：

```css
.chart {
  height: 300px;
}
```

3. 在 `onMounted` 后初始化图表。
4. 数据为空时显示暂无数据。

---

## 20.10 删除数据失败

### 问题表现

删除学生、课程、班级时失败。

### 可能原因

1. 数据存在关联记录。
2. 后端做了关联校验。
3. 外键约束阻止删除。

### 解决方案

这是合理情况。可以：

1. 先删除关联数据。
2. 或改为禁用状态。
3. 或在前端提示“该数据存在关联记录，不能删除”。

# 28. 建议前端请求配置

建议创建：

```text
frontend/src/api/request.js
```

基础配置：

```js
import axios from 'axios'
import { ElMessage } from 'element-plus'

const request = axios.create({
  baseURL: '/api',
  timeout: 10000
})

request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

request.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code !== 200) {
      ElMessage.error(res.message || '请求失败')
      return Promise.reject(res)
    }
    return res.data
  },
  error => {
    ElMessage.error(error.message || '网络异常')
    return Promise.reject(error)
  }
)

export default request
```
